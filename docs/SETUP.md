# Setup & Troubleshooting

## Recommended: loadstring (one URL, auto-updates)

Paste ONCE in your executor (Delta / Codex / Fluxus / etc.):

```lua
loadstring(game:HttpGet("https://raw.githubusercontent.com/OpenJawCode/bed-wars-script/main/loader.lua"))()
```

A green ⚡ button appears in the top-right corner. Tap it to open the menu.

**Every time the script is updated on GitHub, your next paste uses the latest version. No copy-paste of 150 KB every time.**

## Backup: single-file (only if loadstring fails)

If the loadstring shows a red ⚠ BOOT FAILED box (your executor blocks `HttpGet` to GitHub), use the single-file at [`bw-singlefile.lua`](bw-singlefile.lua). Copy the entire file (156 KB / 4443 lines) and paste into your executor.

The single-file is generated by `scripts/build_singlefile.py` from the multi-file source.

## If you see a red ⚠ BOOT FAILED box

The script now shows a visible error overlay if anything goes wrong. Read the error message — it tells you what went wrong.

### Common errors + fixes

| Error | Cause | Fix |
|---|---|---|
| `HttpGet blocked` or empty source | Your executor's `HttpGet` is sandboxed or rate-limited | Use the single-file fallback (paste `bw-singlefile.lua` directly) |
| `HttpGet is not a function` | Old executor without `game:HttpGet` | Update your executor (Delta ≥ 2.0, Codex ≥ latest) |
| `gethui is not a function` | Old executor without `gethui()` | Update your executor |
| `Hookmetamethod is not a function` | Executor doesn't support metamethod hooking | Spy feature disabled automatically; everything else works |
| `Drawing.new is not a function` | Old executor without Drawing API | Update your executor (Delta ≥ 1.5) |
| `Knit bootstrap timeout` | Bedwars still loading | Wait 30s after joining the game, then re-execute |
| `attempt to index nil` | A module failed to load | Re-execute, or use single-file |
| `Unknown error` | Something else | Open executor console (F9 / log panel) — full stack trace is there |

If the BOOT FAILED box says something you don't understand, open the executor's log panel (F9 in Roblox Studio, or the executor's log) — the full error stack trace is printed there with `[bw-script]` prefix.

## Self-test (debug if it's broken)

If the loadstring appears to load but the script doesn't work, run this in the executor's console:

```lua
-- Test 1: Can the executor even HttpGet GitHub?
local ok, source = pcall(function()
  return game:HttpGet("https://raw.githubusercontent.com/OpenJawCode/bed-wars-script/main/loader.lua", true)
end)
print("Test 1 - HttpGet ok:", ok, "source length:", source and #source or "nil")

-- Test 2: Does gethui() work?
local ok2, hui = pcall(function() return gethui() end)
print("Test 2 - gethui ok:", ok2, "hui:", hui and "exists" or "nil")

-- Test 3: Does Drawing.new work? (required for ESP)
local ok3, draw = pcall(function() return Drawing.new("Square") end)
print("Test 3 - Drawing ok:", ok3)

-- Test 4: Is debug introspection available?
local ok4, val = pcall(function() return debug.getupvalue end)
print("Test 4 - debug.getupvalue ok:", ok4)
```

If any test returns `false` (the ok is false), that function is blocked in your executor.

## Manual load (debug mode)

If you want to see exactly what's happening during boot, replace the one-line loadstring with:

```lua
local ok, source = pcall(function()
  return game:HttpGet("https://raw.githubusercontent.com/OpenJawCode/bed-wars-script/main/loader.lua", true)
end)
print("Loader ok:", ok, "source length:", source and #source or "nil")
if ok and source then
  local fn = loadstring(source, "loader")
  if fn then
    print("Loader compiled OK, executing...")
    fn()
  else
    print("Loader parse error")
  end
end
```

This prints each step to the executor console so you can see where it fails.

## How the build works

```
src/                       ← multi-file source (this is what you edit)
├── ui/library.lua
├── features/killaura.lua
└── ...

        ↓  python3 scripts/build_singlefile.py  ↓

docs/bw-singlefile.lua     ← paste this into your executor (backup only)
                           (no HttpGet needed)
```

The build script inlines each module as an IIFE, registers the return value into a shared `_BW` package table, then runs the (slightly stripped) `main.lua` at the end. Total: 28 modules inlined, 4443 lines, 156 KB.

## How the boot works (loadstring)

```
loader.lua  (loadstring target)
  └─ HttpGet main.lua
       └─ HttpGet each src/ module
            └─ register in _BW package table
                 └─ main.lua uses _BW.X
                      └─ Library:CreateWindow()
                           └─ FAB (⚡) appears top-right
```

If any `HttpGet` fails, the loader shows a visible BOOT FAILED box pointing to the single-file backup.

## Debug checklist

1. ✅ Are you in the Bedwars game (not main menu)?
2. ✅ Did the executor actually execute? (Check the executor's log for `[bw-loader]` or `[bw-script]` lines)
3. ✅ Is the green ⚡ visible in the top-right corner? (If yes, the script is running — tap it)
4. ✅ If you see a red ⚠ BOOT FAILED box, read the error message and check the troubleshooting table above
5. ✅ Still broken? Use the single-file version (paste `docs/bw-singlefile.lua` directly)

## Privacy / Permissions

The script needs:
- `game:GetService("Players")` — to read your character + team
- `game:GetService("Workspace")` — to find enemies, beds, items
- `game:GetService("ReplicatedStorage")` — to call the game's remotes (Knit)
- `game:HttpGet` — to fetch modules (loadstring mode only — single-file mode needs no internet)
- `debug.getupvalue` / `debug.getconstants` / `debug.getproto` — to extract remote names from Knit
- `hookmetamethod` — for the Spy feature only (degrades gracefully if missing)

It does NOT:
- Send your data anywhere
- Access your account
- Modify your Roblox client outside the game session
- Persist after the executor ends the session
